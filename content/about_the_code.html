About My Code

 The repositories I'm posting on GitHub are less "useful open-source tools for others" and more "here's some stuff I wrote to test a concept I was learning and I hope it's instructive for others" type repositories. As such, they're not exactly robust or feature-rich. They're more samples and snippets. As I polish up and document some of the projects I'm working on in my own time, I'll begin posting functional pieces of them to public repositories and hopefully they'll be a bit more useful for at least a few people. Until then though, anyone whose curious about compilers, multi-threaded processing, or evaluating the performance of differnt approaches to a given problem should feel free to check out my code.

The Repositories

  • C Declarator Parser

     I was reading the book "Expert C Programming: Deep C Secrets" by Peter Van Der Linden (which I actually highly recommend to anyone who actually wants to work in C or C++ professionally), and I happened upon a programming challenge in the 3rd chapter of the book to write a C declarator parser to make sense of some of the more convoluted C declarations made possible by the standard. Given that I hadn't actually written much in C (having only read a few books on it to wrap my mind around it's nuances before getting my feet wet and getting practiced), this actually seemed like a perfect "Hello, World" type project for me. It would get me a bit more comfortable with it, while helping me really internalize and understand C declarators. My implementation ended up being a fair bit less elegant than Mr. Van Der Linden's, but I'm pleased with it (while still also being open to feedback).

  • Fixed Point Diagnostic Test

     This code is incredibly simple and meant to be instructive more than anything else. It's the first part of a project I'm working on to port the Doom source code over to the .NET CLR in C#. I was looking through their fixed point arithmetic implementation and figured it was senseless to port that code since hardware today is optimized to perform floating point arithmetic. Then I became curious about how optimized hardware had become. Then I decided that, since I'd need to use diagnostic timing to optimize my code eventually anyway, I may as well play around with a trivial case. An hour later, this test driver showed the extent to which native floating point outperformed a simple 16.16 fixed point implementation. It amounted to a 30% - 60% speed increase over sufficiently large test cases.

  • Multi-Threading Demo

     This small project was once again written for instructive purposes. I was having a somewhat illuminating discussion on the distinction between asynchronous processing and multi-threaded processing with a friend and realized I didn't see the dividing line as clearly as I thought. Combing through an old Operating Systems text book I found the chapters about threading, concurrency, and deadlock and decided to write a simple implementation of a thread processor.

  • ToyC Compiler

     Originally done for my Compilers class at VCU, this is a full compiler that will take source code written in a fairly small (though computationally complete) subset of the C programming language and compile it into instructions for the Jasmine assembly language for the Java Virtual Machine. I've since lost the abstract syntax definition, but it can be pretty easily discerned from the "abstractSyntax" directory in the compilers/toyc/src folder.

     The bulk of the work is done in the compilers/toyc/src/parser directory, where the lexer and parser both reside. The interfaces and a few utility files were provided by the instructor, but the rest of the code is mine. I should probably also note that it is, in its current state, technically only 2/3 of a compiler since the lexer and parser both work just fine (though I never did get the parser to pretty-print the AST quite right), but the code generator code is still bugged. I intend to finish this at some point, but my other projects currently keep me from doing so.

     This is the repository for this very website. It's simple, lightweight, and functional, which is exactly and exclusively what I want from a blog site anyway. The least interesting of my repos, but necessary for this site to exist, so there it is.
